>> В категории должна быть возможность пользователей подписываться на рассылку новых статей в этой категории.
Возможность добавлена на странице поиска и на персональной странице пользователя.

>> Если пользователь подписан на какую-либо категорию, то, как только в неё добавляется новая статья, её краткое содержание приходит пользователю на электронную почту, которую он указал при регистрации. В письме обязательно должна быть гиперссылка на саму статью, чтобы он мог по клику перейти и прочитать её.
Данная возможность реализована. Отправка почты запускается в отдельном потоке, чтобы не замедлять работу приложения.

>> Если пользователь подписан на какую-либо категорию, то каждую неделю ему приходит на почту список новых статей, появившийся за неделю с гиперссылкой на них, чтобы пользователь мог перейти и прочесть любую из статей.
Реализовано с помощью django_apscheduler

>> Добавьте приветственное письмо пользователю при регистрации в приложении.
Приходит письмо с ссылкой на активацию. 

>> Один пользователь не может публиковать более трёх новостей в сутки.
Реализовано. в файл настроек добавлена переменная MAX_POST_PER_DAY = 3, с помощью которой можно задать лимит.


Изменения по безопасности:
Сделал проверку по пользователям, чтобы нельзя было редактировать/удалять сообщения других пользователей.
При попытке изменить скрытые данные в форме редактирования (input type="hidden" name="author" value=...) , также будет выдана ошибка.
